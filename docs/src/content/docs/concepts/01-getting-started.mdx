## Installation

```bash
npm install @requence/tokenized-search
```

All UI dependencies (`react`, `react-dom`, the `@tiptap/*` packages, and
`tailwind-merge`) are **optional peer dependencies**, so installing the package
pulls in none of them — a server that only parses queries via the
[`core` entry](/reference/02-parser-utilities/) stays free of UI packages. To
use the `<TokenizedSearch />` component, install them alongside:

```bash
npm install react react-dom tailwind-merge \
  @tiptap/core @tiptap/pm @tiptap/react \
  @tiptap/extension-document @tiptap/extension-history \
  @tiptap/extension-paragraph @tiptap/extension-text
```

## Tailwind CSS Setup

This package uses [Tailwind CSS](https://tailwindcss.com/) utility classes internally (via `tailwind-merge`). Add the `@source` directive to your CSS entry point so Tailwind scans the package for class names:

```css
@import "tailwindcss";
@source "../node_modules/@requence/tokenized-search";
```

## Basic Usage

```tsx
import { TokenizedSearch } from '@requence/tokenized-search'

function App() {
  return (
    <TokenizedSearch
      tokens={[
        {
          key: 'status',
          label: 'Status',
          options: [
            { value: 'active', label: 'Active' },
            { value: 'inactive', label: 'Inactive' },
          ],
        },
        {
          key: 'source',
          label: 'Source',
          options: async (query, signal) => {
            const res = await fetch(`/api/sources?q=${query}`, { signal })
            return res.json()
          },
        },
      ]}
      onSearch={(segments, rawText) => {
        console.log(segments, rawText)
      }}
    />
  )
}
```

The component works out of the box with default styling. To customize the appearance, use [slot sub-components](/concepts/04-styling/).

## Controlled vs Uncontrolled

**Uncontrolled** (default) — pass `defaultValue` to set the initial query:

```tsx
<TokenizedSearch defaultValue="status:active" ... />
```

**Controlled** — pass `value` and `onChange` to manage state externally:

```tsx
const [value, setValue] = useState('')

<TokenizedSearch
  value={value}
  onChange={(newValue, segments) => setValue(newValue)}
  ...
/>
```

## Auto-commit

By default the search only commits when the user presses Enter or clicks the
submit button. Pass `autoCommit` to fire `onSearch` on every change instead —
with the technical query, once any async options have resolved — and hide the
submit button. Useful when the query should apply immediately without a manual
step:

```tsx
<TokenizedSearch
  autoCommit
  tokens={tokens}
  onSearch={(segments, rawText) => applyFilter(rawText)}
/>
```
