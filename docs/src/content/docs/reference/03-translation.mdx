Translation utilities convert between display-form queries (using labels) and technical-form queries (using keys/values).

## toTechnicalQuery

Convert a display-form query string to its technical counterpart.

```tsx
import { toTechnicalQuery } from '@requence/tokenized-search'

const technical = toTechnicalQuery('Status:Active', tokens)
// ‚Üí 'status:active'
```

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `rawText` | `string` | ‚Äî | Display-form query |
| `tokens` | `TokenizedSearchTokenDefinition[]` | ‚Äî | Token definitions |
| `negationLabel` | `string` | `'not'` | Negation prefix |
| `optionValueMap` | `Map<string, Map<string, string>>` | ‚Äî | Optional label‚Üívalue cache for resolving async token values |

### Returns

`string` ‚Äî the technical-form query.

## toDisplayQuery

Convert a technical-form query string to display form.

```tsx
import { toDisplayQuery } from '@requence/tokenized-search'

const display = toDisplayQuery('status:active', tokens)
// ‚Üí 'Status:Active'
```

### Parameters

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `technicalText` | `string` | ‚Äî | Technical-form query |
| `tokens` | `TokenizedSearchTokenDefinition[]` | ‚Äî | Token definitions |
| `negationLabel` | `string` | `'not'` | Negation prefix |

### Returns

`string` ‚Äî the display-form query.
